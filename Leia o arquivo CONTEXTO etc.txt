Integracao com fluxo de conclusao (Fase 3):

Quando o aluno conclui um bloco e informa o assunto (ex: "Balanco Patrimonial"), o sistema busca no edital se existe um topico/subtopico correspondente para aquela materia.
Se encontrar match (busca por nome similar/contido), marca automaticamente como 'concluido' no edital_progresso.
Se nao encontrar match exato, sugere topicos do edital para o aluno selecionar (dropdown com autocomplete ao informar o assunto na Etapa 1 da conclusao, filtrado pelos topicos do edital daquela materia).
Questoes feitas/corretas tambem sao agregadas no edital_progresso por topico.

Professor configura estrutura do edital (materias -> topicos -> subtopicos) dentro do Plano. Pode importar via Excel.

Fase 6: Evolucao Automatica do Ciclo
Arquivos: js/evolucao.js
Schema Supabase:
sqlCREATE TABLE ciclo_historico (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES auth.users(id),
  plano_id UUID REFERENCES planos(id),
  numero_ciclo INTEGER NOT NULL DEFAULT 1,
  blocos JSONB NOT NULL DEFAULT '[]',
  questoes_resumo JSONB DEFAULT '{}',
  completed_at TIMESTAMPTZ DEFAULT now()
);
Tipos de gatilho:

ciclos_completos: apos N ciclos -> acao
progresso_edital: materia X atingiu Y% -> acao
prerequisito: materia X concluida ate Z% -> desbloqueia materia Y

Tipos de acao:

adicionar_materia: inclui nova materia no ciclo
alterar_peso: muda peso de materia existente
desbloquear_materia: adiciona materia bloqueada
remover_materia: retira materia do ciclo

Fluxo ao completar ciclo:

Salvar snapshot em ciclo_historico
Avaliar todas as regras
Mostrar resumo das regras ativadas
Gerar novo ciclo com alteracoes aplicadas
Resetar blocos para o novo ciclo

Configuracao: Professor define regras no Plano. Aluno pode ver regras + adicionar proprias na aba Configuracoes.
